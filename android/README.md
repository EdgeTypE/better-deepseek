# Better DeepSeek — Android (Phase 1)

Wraps the BDS web extension in a single-Activity Android WebView. The Svelte UI,
parsers, tools, and injection logic are reused 1-for-1; only the host-specific
APIs (storage, fetch, asset URLs) are replaced by a native bridge.

This directory only contains the native shell. Build the JS bundle from the
repo root first, then build the APK from here.

## Required system dependencies

Install before building from source:

| Tool          | Version       | Notes |
|---------------|---------------|-------|
| Node.js       | >= 18         | Already required for the extension build. |
| JDK           | 17            | Set `JAVA_HOME` to a JDK 17 install. Adoptium Temurin works. |
| Android SDK   | API 34        | Easiest via Android Studio (`Tools -> SDK Manager`). |
| Build-tools   | 34.0.0        | Installed alongside the SDK. |
| Gradle        | 8.7           | Only needed once to bootstrap the wrapper (see below). |

Required `local.properties` (auto-generated by Android Studio, or create manually):

```
sdk.dir=C:\\Users\\<you>\\AppData\\Local\\Android\\Sdk
```

## One-time wrapper bootstrap

The wrapper jar/scripts (`gradlew`, `gradlew.bat`, `gradle/wrapper/gradle-wrapper.jar`)
are not checked in. Run this once after cloning, from inside `android/`:

```
gradle wrapper --gradle-version 8.7 --distribution-type bin
```

After that, `./gradlew` is self-contained.

## Build flow

```
# 1. Build and stage the JS bundle.
npm run build:android

# 2. Build the debug APK.
cd android
./gradlew assembleDebug
```

The APK lands at `android/app/build/outputs/apk/debug/app-debug.apk`.

## What Phase 1 ships

- Full-screen WebView loading `chat.deepseek.com`.
- `AndroidBridge` `@JavascriptInterface` exposing storage, generic fetch, asset URLs.
- `chrome.*` JS polyfill in `src/platform/android-chrome-polyfill.js` that routes
  storage/runtime/getURL through the bridge — extension code is unchanged.
- WebViewAssetLoader serves the BDS bundle under `https://bds-asset.local/bds/...`.

Phase 2 will flesh out blob downloads, voice input gating, and folder upload UX.
